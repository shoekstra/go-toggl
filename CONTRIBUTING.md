# Contributing

Contributions are welcome! Please follow these guidelines:

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/my-feature`

## Making Changes

1. Make your changes
2. Run tests: `task test`
3. Run linter: `task lint`
4. Format code: `task fmt`
5. Use conventional commits: `git commit -m "feat: add new feature"`

## Pull Requests

1. Keep PRs focused and atomic
2. Include tests for new functionality
3. Update documentation as needed
4. Reference any related issues

## Code Style

- Follow Go conventions
- Use table-driven tests
- Add godoc comments to public items
- Run `task fmt` before committing
